Added `withDefaultStyleSlot(Component, slotKey)` (`utils/with-default-style-slot`) so any component can subscribe to the api-maker default-style context the same way `Text` does. Shipped a slot-aware `TextInput` (`utils/text-input`) using the helper, since RN's built-in `TextInput` does not inherit the `Text` slot. `WithDefaultStyle` no longer requires slot keys to be pre-registered, and `useMergedStyle` treats a missing slot default as an empty list, so consumers can introduce their own slots (e.g. `Link`) by composing the helper around their underlying component. The pure merge step is now exported as `computeSlotStylesList` and covered by tests.
